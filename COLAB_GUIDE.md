# Руководство по сборке Android APK в Google Colab

Это руководство поможет вам собрать Android APK используя Google Colab без необходимости локальной установки Buildozer и Android SDK.

## Преимущества использования Google Colab

- ✅ Не требует локальной установки Buildozer
- ✅ Не требует установки Android SDK/NDK локально
- ✅ Работает на любой платформе (Windows, Mac, Linux) через браузер
- ✅ Бесплатно
- ✅ Автоматизированная установка всех зависимостей
- ✅ Готовый APK можно скачать одним кликом

## Быстрый старт

### Шаг 1: Откройте notebook в Google Colab

1. Откройте [Google Colab](https://colab.research.google.com/)
2. Нажмите **File → Upload notebook**
3. Загрузите файл `build_android_colab.ipynb`
4. Или откройте напрямую из GitHub (если репозиторий публичный):
   - File → Open notebook → GitHub
   - Введите URL репозитория

### Шаг 2: Запустите все ячейки

1. Нажмите **Runtime → Run all** (или `Ctrl+F9`)
2. Или запускайте ячейки по порядку (Shift+Enter)

### Шаг 3: Дождитесь завершения сборки

- Первая сборка займет **30-60 минут** (скачивание SDK/NDK)
- Последующие сборки будут быстрее (если не перезапускать runtime)

### Шаг 4: Скачайте APK

- APK автоматически скачается в последней ячейке
- Или найдите файл в папке `bin/` и скачайте вручную

## Подробное описание ячеек

### Ячейка 1: Установка зависимостей

Устанавливает:
- Системные пакеты: Java JDK, build tools, autotools
- Python пакеты: buildozer, cython

**Время выполнения:** 2-5 минут

### Ячейка 2: Загрузка проекта

Два варианта:

#### Вариант A: Клонирование из GitHub (рекомендуется)

```python
!git clone https://github.com/suninvest777/mallware.git
%cd mallware/ready_app
```

#### Вариант B: Загрузка файлов вручную

1. В Colab нажмите на иконку папки слева (Files)
2. Нажмите "Upload" и загрузите папку `ready_app/`
3. Раскомментируйте код в ячейке "Вариант B"
4. Укажите правильный путь к загруженным файлам

### Ячейка 3: Применение патча для pyjnius

Применяет патч для исправления проблем совместимости Python 2/3 в pyjnius.

**Что делает:**
- Находит файл `jnius_utils.pxi` в установленном pyjnius
- Заменяет все вхождения `long` на `int` (Python 3 совместимость)

### Ячейка 4: Инициализация Buildozer

1. Принимает лицензии Android SDK автоматически
2. Инициализирует Buildozer (скачивает базовый SDK)
3. Устанавливает cmdline-tools если нужно
4. Создает симлинки для sdkmanager

**Время выполнения:** 5-10 минут

### Ячейка 5: Настройка Android SDK

Устанавливает необходимые компоненты:
- build-tools (34.0.0, 33.0.0 или 32.0.0)
- platform-tools
- platforms;android-33

**Время выполнения:** 5-10 минут

### Ячейка 6: Применение патча и проверка .c файлов

1. Применяет патч для pyjnius (если pyjnius уже установлен)
2. Проверяет наличие `.c` файлов Cython
3. Генерирует `.c` файлы если они отсутствуют

### Ячейка 7: Сборка APK

**ВНИМАНИЕ:** Это самая долгая ячейка!

**Время выполнения:** 30-60 минут (первая сборка)

**Что происходит:**
- Buildozer скачивает Android NDK (если еще не скачан)
- Компилирует Python для Android
- Компилирует все зависимости (kivy, pyjnius, requests)
- Собирает APK

**Логи:** Все логи сохраняются в `/tmp/buildozer.log`

### Ячейка 8: Скачивание APK

Автоматически:
1. Ищет APK файл в папке `bin/`
2. Показывает информацию о файле (размер, путь)
3. Скачивает APK через Colab API

## Решение проблем

### Проблема: "Buildozer не найден"

**Решение:**
```python
!pip3 install --user buildozer
import os
os.environ['PATH'] = os.environ['HOME'] + '/.local/bin:' + os.environ['PATH']
```

### Проблема: "SDK не найден" или "sdkmanager не найден"

**Решение:**
1. Перезапустите ячейку 4 (Инициализация Buildozer)
2. Проверьте, что лицензии приняты
3. Убедитесь, что cmdline-tools установлены

### Проблема: "Ошибка компиляции pyjnius" или "long не найден"

**Решение:**
1. Убедитесь, что патч применен (ячейка 3 и 6)
2. Проверьте, что Cython установлен: `!pip3 show cython`
3. Попробуйте перезапустить runtime и начать заново

### Проблема: "APK не найден"

**Решение:**
1. Проверьте логи сборки: `!tail -100 /tmp/buildozer.log`
2. Убедитесь, что сборка завершилась успешно (код выхода 0)
3. Проверьте папку `bin/` вручную: `!ls -la bin/`
4. Возможно, APK находится в другом месте: `!find . -name "*.apk"`

### Проблема: "Timeout" или "Runtime disconnected"

**Решение:**
- Colab отключает runtime после 90 минут неактивности
- Если сборка прервалась, перезапустите с ячейки 7 (Сборка APK)
- Buildozer кэширует скачанные файлы, поэтому повторная сборка будет быстрее

### Проблема: "Недостаточно места на диске"

**Решение:**
- Colab предоставляет ~15GB дискового пространства
- Очистите кэш: `!rm -rf ~/.buildozer/android/platform/android-sdk/.cache`
- Или перезапустите runtime (Runtime → Restart runtime)

## Оптимизация времени сборки

### Использование кэша

Buildozer кэширует:
- Android SDK/NDK (после первой загрузки)
- Скомпилированные зависимости

**Совет:** Не перезапускайте runtime между сборками, чтобы сохранить кэш.

### Быстрая пересборка

Если нужно пересобрать APK после изменения кода:
1. Загрузите обновленные файлы
2. Запустите только ячейку 7 (Сборка APK)
3. Сборка займет 5-15 минут (без скачивания SDK/NDK)
